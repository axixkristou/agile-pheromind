Introduction to AgilePheromind

AgilePheromind is an artificial intelligence (AI) system designed as a collaborative swarm of specialized AI agents, powered by the Pheromind framework. Its primary purpose is to assist and enhance the capabilities of software development teams following Agile methodologies.

Unlike monolithic AI tools, AgilePheromind functions as an ecosystem where specialized agents interact indirectly to accomplish complex tasks, learn from the project context, and adapt to the team's needs.

Key Features

  • Swarm intelligence via stigmergy (indirect agent communication)
  • Hierarchical orchestration with workflow scripts
  • Centralized state management with .pheromone file
  • Deep integration with Roo Code platform
  • Model Context Protocol (MCP) servers for external tool integration

Workflow Scripts (01_AI-RUN/*.md)

These Markdown files are the "programs" that the @uber-orchestrator executes. Each script corresponds to a high-level AgilePheromind functionality.

Example Workflow Script Structure

# 01_AI-RUN/start_user_story.md

## Objective
Assist in starting development of a new User Story.

## Key Agents Involved
- @po-assistant
- @developer-agent
- @task-estimator

## MCPs Potentially Used
- Azure DevOps MCP
- Git Tools MCP
- Context7 MCP

## Workflow Phases

### Phase 1: Understand Requirements
**Agent:** @po-assistant
**Inputs:** 
- US ID from parameters
- Current sprint context from .pheromone
**Actions:**
- Use Azure DevOps MCP to get US details
- Analyze acceptance criteria
**Output:** 
- Summary of requirements
- Updated .pheromone with US analysis

### Phase 2: Task Breakdown
**Agent:** @task-estimator
**Inputs:** 
- US analysis from Phase 1
- Similar past tasks from memoryBank
**Actions:**
- Decompose into technical tasks
- Estimate effort
**Output:**
- Task list with estimates
- Updated .pheromone with task breakdown

The Memory Bank: The Learning Core

The memoryBank in .pheromone is more than a simple log. It's a structured knowledge base that enables the system to:

  • Maintain long-term context: Remember past architectural decisions, similar task estimates, and solutions to recurring problems.
  • Learn from interactions: Agents can refine future estimates by comparing past estimates with actual time spent.
  • Provide rich context to agents: Before starting a task, an agent can consult relevant information about the US, related tasks, conventions, etc.
  • Ensure consistency: Coding and design conventions are referenced here, ensuring all agents (and humans) work from the same standards.
  • Track evolution: History of statuses, reviews, and deployments.

Memory Bank Structure

projectContext

Tech stack, links to conventions, ADO project name

userStories

Detailed history, statuses, related tasks, analyses

commits & pullRequests

Tracked code changes and review history

riskRegister

Identified risks and mitigation strategies

Integration with Roo Code

AgilePheromind leverages several advanced Roo Code features:

Task Orchestration

Uses Roo Code's Orchestrator mode (Boomerang) to coordinate complex workflows.

Mode Switching

Agents can switch between modes as needed for different task requirements.

Persistent Models

Each agent can use a different AI model optimized for its specific tasks.

File Access Control

Permissions in .roomodes limit agent access to certain file types for security.
